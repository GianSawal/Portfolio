Active Directory Domain Infrastructure

This project demonstrates the deployment and configuration of a Windows Server Active Directory Domain environment used for centralized user management, network authentication, and administrative control within an enterprise network.

Domain Infrastructure Overview

Active Directory is a directory service developed by Microsoft that enables centralized management of users, computers, and network resources within a Windows domain environment. In this project, a Windows Server 2019 system is deployed as a Domain Controller to manage authentication and network policies across the organization.

The Domain Controller stores information about all domain objects including users, groups, and computers. Administrators can use this system to control access to shared resources, enforce security policies, and manage network permissions from a centralized platform.

Within the domain structure, Organizational Units (OUs) are created to separate departments such as administration, marketing, and IT. These organizational units help simplify management by grouping users and devices based on their role within the organization.

Security groups are also configured to control permissions for specific resources. For example, the Marketing department group allows administrators to apply permissions and policies specifically to users within that department.

This centralized authentication system improves network security, simplifies administration, and ensures consistent policy enforcement across all domain-joined devices.

Technologies Used

Windows Server 2019

Windows Server provides the platform for hosting the Active Directory Domain Services role, enabling centralized identity and access management.

Active Directory Domain Services

AD DS allows administrators to manage users, computers, and resources within a domain environment through centralized authentication and authorization services.

Organizational Units (OU)

Organizational Units provide a logical structure within Active Directory that allows administrators to group users and devices based on departments or functions.

Security Groups

Security groups are used to assign permissions to multiple users at once, simplifying access control for shared network resources.

DNS Integration

Active Directory relies on DNS services for domain name resolution, allowing domain controllers and clients to locate services within the network.

Centralized Authentication

Domain users authenticate once and gain access to multiple network resources based on their assigned permissions.
